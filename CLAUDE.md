# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

A Unity 6.0+ Editor plugin written in C# that integrates the Anthropic Claude API to generate game scripts and scene objects directly within the Unity Editor.

## Development Commands

This is a Unity project — there is no CLI build command. All builds, tests, and runs happen inside the Unity Editor.

**Running Tests:**
- Open the Unity project in Unity 6.0+
- Window → General → Test Runner → EditMode tab → Run All
- Tests live in `Assets/Editor/Tests/`

**Installing the plugin (manual):**
- Copy `Assets/Editor/ClaudeAssitant/` into another Unity project's `Assets/Editor/` folder
- Set your API key via the Assistant window → Settings

**Exporting as Unity Package:**
- Right-click `Assets/Editor/ClaudeAssitant/` in Unity → Export Package

## Architecture

The plugin follows a layered architecture with the Strategy pattern for generation modes.

```
UI Layer (EditorWindow)
    └── ConversationController  (orchestrator/pipeline)
            ├── ClaudeApiClient         (HTTP + streaming)
            ├── ConversationHistory     (context window management)
            ├── IntentClassifier        (Script vs Scene detection)
            └── IGenerationHandler      (strategy interface)
                    ├── ScriptGenerationHandler   → saves .cs files
                    └── SceneGenerationHandler    → builds GameObjects in scene
```

**Key files:**

| File | Responsibility |
|------|---------------|
| `Assets/Editor/ClaudeAssitant/UI/ClaudeAssistantWindow.cs` | Pure UI — EditorWindow, renders chat, delegates all logic |
| `Assets/Editor/ClaudeAssitant/Core/ConversationController.cs` | Orchestrates the full generation pipeline per user message |
| `Assets/Editor/ClaudeAssitant/Core/ClaudeApiClient.cs` | Singleton HTTP client; handles streaming responses via `UnityWebRequest` |
| `Assets/Editor/ClaudeAssitant/Core/ConversationHistory.cs` | Manages chat state; trims old messages to stay within API context limits |
| `Assets/Editor/ClaudeAssitant/Core/ClaudeConfig.cs` | ScriptableObject for config; API key stored in `EditorPrefs`, never serialized |
| `Assets/Editor/ClaudeAssitant/Handlers/IGenerationHandler.cs` | Strategy interface — implement this to add new generation modes |
| `Assets/Editor/ClaudeAssitant/Handlers/ScriptGenerationHandler.cs` | Writes generated C# files to `Assets/Scripts/Generated/` |
| `Assets/Editor/ClaudeAssitant/Handlers/SceneGenerationHandler.cs` | Spawns GameObjects in the active scene using `Undo.RegisterCreatedObjectUndo` |
| `Assets/Editor/ClaudeAssitant/Utils/CodeExtractor.cs` | Parses single and multi-script markdown blocks from API responses |
| `Assets/Editor/ClaudeAssitant/Utils/IntentClassifier.cs` | Keyword-based classifier; returns Script/Scene/Unknown with tie-breaking |
| `Assets/Editor/ClaudeAssitant/Models/GenerationResult.cs` | Factory result type (Ok/Fail) — used throughout the pipeline |
| `Assets/Editor/Tests/IntentClassifierTests.cs` | NUnit EditMode tests for intent classification |

## Key Design Decisions

- **API key security:** Stored only in `EditorPrefs` (per-machine, not in project files). Never write it to ScriptableObjects or serialized assets.
- **Multi-script support:** `CodeExtractor` parses multiple fenced code blocks from a single response. Each block with a `// ClassName.cs` comment becomes a separate file.
- **Undo support:** All scene modifications use Unity's `Undo` system so changes are reversible with Ctrl+Z.
- **Domain reload persistence:** Chat history survives Unity Editor domain reloads via `PersistedMessage` (serializable DTOs stored in `SessionState`).
- **Adding a new generation mode:** Implement `IGenerationHandler`, register it in `ConversationController`, and add keywords to `IntentClassifier`.

## Important Notes

- The folder is named `ClaudeAssitant` (typo, missing an 's') — keep this name to avoid breaking existing installations.
- `Assets/Scripts/Generated/` contains example scripts generated by the tool itself — they are demo output, not plugin source code.
- The `ClaudeAssitant/` folder at the repo root is a source mirror of `Assets/Editor/ClaudeAssitant/` used for distribution; keep both in sync when making changes.
